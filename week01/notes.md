### 3. Git Basics

Learned the purpose of:

* `git status`
* `git add`
* `git check-ignore -v`

I also understood the basic Git workflow:

```
Create file
    ↓
git status
    ↓
git add
    ↓
git commit
    ↓
git push
```

---

### 4. `.gitignore`

Learned why `.gitignore` is necessary.

The `.gitignore` inside `.venv` only ignores files inside the virtual environment, while the project root `.gitignore` tells Git which files and directories should not be tracked for the entire project.

---

### 5. `pyproject.toml`

I learned that `pyproject.toml` is the central configuration file of a modern Python project.

It defines information such as:

* Project name
* Project version
* Required Python version
* Dependencies

This file is intended to be read and updated by both developers and tools like `uv`.

---

### 6. `uv.lock`

I learned that `uv.lock` is automatically generated by `uv`.

It records the exact versions of all installed dependencies so that everyone working on the project gets the same environment.

I should normally **not edit this file manually**.

---
