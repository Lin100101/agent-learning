# 2026 暑假双 Agent 项目与实习倒排计划

## 最终目标

在 2026 年 8 月底前完成两个可部署、可测试、可讲解的 Agent 项目，并开始投递杭州中小厂的 AI 应用开发相关实习岗位。

两个项目形成一条完整故事线：

```text
电商售后客服 Agent
→ 执行真实业务任务
→ AgentTrace 记录、分析和评测执行过程
```

---

## 项目一：电商售后客服 Agent

### 项目定位

面向电商售后场景的业务 Agent。它不仅回答问题，还能够查询订单、判断退款条件、创建工单并在高风险场景升级人工处理。

### MVP 功能

- FastAPI 对话接口；
- 订单查询工具；
- 退款资格判断工具；
- 售后知识检索；
- 工单创建工具；
- 会话状态保存；
- 高风险或低置信度场景人工升级；
- 参数验证、结构化错误和最大循环保护；
- 基础运行 Trace；
- 自动化测试和评测用例。

### 验收标准

- 至少实现 4 个真实业务工具；
- 支持至少 3 个需要连续工具调用的任务；
- 准备不少于 20 条测试用例；
- 非法参数和工具异常不会导致服务崩溃；
- README 包含架构、运行方式、示例和已知限制；
- 提供演示接口或演示视频。

---

## 项目二：AgentTrace 追踪与自动评测平台

### 项目定位

用于记录 Agent 决策过程、分析工具调用质量并批量评测 Agent 版本的平台。

### MVP 功能

- 接收 Agent 运行 Trace；
- 保存任务、决策轮次、工具调用、参数、结果和错误；
- Trace 列表和详情页面；
- 成功率、耗时和工具调用次数统计；
- 评测数据集管理；
- 批量执行测试任务；
- Prompt 或 Agent 版本对比；
- 失败任务回放；
- 接入项目一的客服 Agent。

### 验收标准

- 能完整展示一次多步 Agent 执行轨迹；
- 能按成功、失败、工具名称等条件筛选；
- 能批量运行不少于 20 条评测用例；
- 能比较至少两个 Agent 或 Prompt 版本；
- 前后端均可启动，核心页面可演示；
- README 包含架构图、数据模型、截图和运行方式。

## 每日时间建议

如果每天可投入 3—4 小时：

```text
2 小时      Agent 学习与项目开发
45 分钟     算法题
30 分钟     Python、数据库、网络等面试基础
15—30 分钟 复盘、笔记和 Git 提交
```

学习内容应尽快进入项目，避免长时间只做孤立 Demo。每周至少形成一个可以运行、测试或展示的成果。
