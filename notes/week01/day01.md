### 2. 项目结构

创建了 AI Agent 学习项目的基础目录结构：

```text
agent-learning/
├── .venv/
├── docs/
├── notes/
├── week01/
├── README.md
├── ROADMAP.md
├── pyproject.toml
├── uv.lock
└── LICENSE
```

认识到，一个结构清晰、组织规范的项目，比把所有 Python 文件都放在根目录下更易于维护和扩展。

---

### 3. Git 基础

学习了以下 Git 命令的作用：

* `git status`
* `git add`
* `git check-ignore -v`

同时理解了 Git 最基本的工作流程：

```text
创建或修改文件
        ↓
git status
        ↓
git add
        ↓
git commit
        ↓
git push
```

开始建立版本控制的基本思维，而不仅仅是保存代码。

---

### 4. `.gitignore`

学习了 `.gitignore` 文件的作用。

了解到：

* `.venv` 目录中的 `.gitignore` 仅用于忽略虚拟环境内部的文件；
* 项目根目录下的 `.gitignore` 才是真正告诉 Git 整个项目哪些文件或目录不需要纳入版本控制。

这也是每个 Python 项目的基本配置之一。

---

### 5. `pyproject.toml`

学习了 `pyproject.toml` 的作用。

它是现代 Python 项目的核心配置文件，主要负责管理：

* 项目名称
* 项目版本
* Python 版本要求
* 项目依赖
* 其他开发配置

开发者和工具（如 `uv`）都会读取并维护该文件。

---

### 6. `uv.lock`

学习了 `uv.lock` 的作用。

该文件由 `uv` 自动生成，用于记录项目所有依赖的**精确版本**，保证不同开发环境安装得到完全一致的依赖。

通常情况下：

* 不需要手动修改；
* 应交由 `uv` 自动维护。

---

学习了几个常用快捷键：

这些快捷键将成为日常开发过程中最常使用的操作。

## 今日最大收获

今天并没有正式开始编写 Python 代码。

最大的收获是学习了如何像一名软件工程师一样搭建开发环境，并理解项目中每个文件和工具存在的意义。

相比于"会创建这些文件"，更重要的是知道：

**为什么需要这些文件，它们分别解决了什么问题。**
